# 人工批量评价操作手册

本文档记录已经验证过的卖家工作台批量评价路径。它是历史订单人工维护时的操作手册，不是默认自动化提交路径。

## 使用条件

只在同时满足以下条件时使用：

- 账号 Cookie 有效，且 `./scripts/xianyu-service.sh doctor` 显示长期运行的 live/web 服务仍在主 checkout。
- 用户明确要求对历史交易订单批量评价。
- 平台页面能确认存在 `待卖家评价` 订单。
- 批量评价文案已经确认，例如 `交易愉快`。

遇到登录失效、滑块、验证码、风控或权限错误时停止操作并记录原因，不做绕过。

## 工作台入口

当前鱼小铺卖家工作台的评价管理路由是：

```text
https://seller.goofish.com/?site=COMMONPRO#/seller-trade/evaluation-manage
```

如果直接打开没有渲染到目标页面，先进入卖家工作台首页，再点击侧边栏 `交易` -> `评价管理`。页面可能弹出 `PC端工作台使用反馈收集` 弹窗，可用 `Escape` 或关闭/取消按钮先关闭，避免遮挡表格。

目标页面应出现 `全部`、`待卖家评价` 计数，订单表格，行复选框，单行 `评价` 操作，以及底部 `批量评价` 按钮。

## 批量操作步骤

1. 用已登录的浏览器上下文打开评价管理路由。
2. 核对 `待卖家评价` 数量和当前页条数；已验证页面默认每页 20 条。
3. 勾选表头复选框，选择当前页可见订单。
4. 确认底部显示 `已选 (20/20)` 或预期选择数量。
5. 点击 `批量评价`。
6. 在弹窗中确认标题显示 `评价（已选中...条订单记录）`。
7. 保持默认 `好评`，除非用户明确要求其他评价等级。
8. 在文本框填写已确认文案。
9. 点击 `发布`。
10. 等待平台请求和页面状态稳定后再关闭浏览器。

## 结果校验

优先使用平台返回结果，而不是只看 DOM 文本。卖家工作台页面可能残留隐藏的 `禁止访问`、反馈弹窗等无关文本。

关键网络响应是：

```text
mtop.taobao.idle.merchant.rate.create
```

成功结果会返回 `successOrderIds`；失败订单会出现在 `failOrderInfos`。如果失败原因为 `超出30天的订单不允许评价`，这是平台永久限制，不应作为自动化失败反复重试。

提交后重新刷新或查询评价管理列表，`待卖家评价` 数量应按成功订单数减少。

## 本地状态

`data/` 下的 SQLite 数据是运行状态，禁止提交。如果本地 `review_tasks` 中存在平台已经确认成功的订单，只能按平台返回的 `successOrderIds` 精确同步为 `submitted`，并记录实际使用的评价文案。不要仅因为订单被选中就标记为已评价。
